# UI QA 체크리스트

이 문서는 회의실 예약 시스템의 화면 검수 기준을 빠르게 확인하기 위한 체크리스트다.

## 공통
- [ ] 모든 주요 화면에서 제목, 설명, 버튼 문구가 서비스 톤으로 보인다.
- [ ] 입력 폼의 간격, 라운드, 포커스 상태가 일관된다.
- [ ] 대비가 낮아 읽기 어려운 텍스트가 없다.
- [ ] 모바일에서 카드와 패널이 세로로 자연스럽게 쌓인다.
- [ ] 탭 키로 주요 버튼과 입력 칸을 이동할 수 있다.

## 홈
- [ ] 첫 화면에 개발용 표현이 남아 있지 않다.
- [ ] 로그인과 회원가입 진입 경로가 명확하다.
- [ ] 핵심 기능 설명이 3개 이하의 카드로 읽힌다.

## 로그인 / 가입
- [ ] 이메일, 비밀번호, 인증번호 입력 라벨이 명확하다.
- [ ] 에러 메시지가 너무 기술적이지 않다.
- [ ] 인증 단계별 이동 경로가 자연스럽다.

## 예약
- [ ] 이번 주 테이블의 열 제목이 읽기 쉽다.
- [ ] 그룹 색상이 과하지 않고 구분만 돕는다.
- [ ] 내 예약 삭제 확인 흐름이 분명하다.

## 관리자
- [ ] 승인, 그룹, 정책, 감사, 정기 예약 화면의 역할이 구분된다.
- [ ] 위험 작업 버튼이 보조 버튼과 명확히 구분된다.
- [ ] 감사 로그와 시리즈 취소 내역을 읽을 수 있다.
