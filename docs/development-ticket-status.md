# 개발 티켓 상태표

이 문서는 현재 저장소와 문서에서 확인 가능한 정보를 바탕으로 한 **잠정 상태표**다.  
명시적인 티켓 트래커가 없기 때문에, 아래 상태는 코드와 문서 기준의 추정 결과이며 수동 업데이트가 필요하다.

기준일: 2026-06-27

## 상태 범례
- `완료`: 개발, 테스트, 반영까지 모두 끝남
- `개발완료`: 구현과 커밋은 끝났지만 테스트/반영 전
- `진행중`: 관련 코드 또는 골격이 일부 존재함
- `미시작`: 문서상 정의만 있고 코드 반영 확인 안 됨
- `미확인`: 현재 저장소만으로 상태 판별이 어려움

## 현재 요약
- 완료: `11`
- 진행중: `5`
- 미시작 또는 미확인: `15`

| 티켓 | 제목 | 상태 | 근거 |
|---|---|---|---|
| T-001 | 프로젝트 초기 세팅 | 완료 | Next.js 앱 구조, Supabase 연동 파일, 기본 스크립트, 배포 가능한 초기 스캐폴딩 존재 |
| T-002 | 서울시간 기준 정책 정의 | 완료 | Supabase migration에 `Asia/Seoul` 기준 검증 로직 존재 |
| T-011 | 이메일 인증 플로우 | 완료 | Supabase `signUp`과 인증 리다이렉트 흐름, confirm route 존재 |
| T-013 | 로그인 및 세션 유지 | 완료 | 로그인 화면과 Supabase 인증, 세션 갱신 흐름 존재 |
| T-031 | 회의실 삭제 시 예약 연쇄 삭제 정책 | 완료 | `bookings`가 `rooms`에 `on delete cascade`로 연결됨 |
| T-040 | 예약 가능 시간대 검증 | 완료 | 마이그레이션에 요일/시간/30분 단위 검증이 존재 |
| T-041 | 1회 최대 1시간 제한 | 완료 | 마이그레이션의 트리거 검증에 포함됨 |
| T-043 | 동일 회의실 중복 예약 방지 | 완료 | `bookings_no_overlap` exclusion constraint 존재 |
| T-050 | 예약 생성 | 완료 | 예약 추가 UI와 DB insert 흐름이 존재 |
| T-060 | 사용자 대시보드: 전체 예약 조회 | 완료 | `/bookings`에서 전체 예약 목록을 조회 가능 |
| T-091 | 관리자 권한 분기 | 완료 | 서버 가드와 RLS가 관리자/승인 사용자 분기를 반영 |

## 운영 메모
- 이 상태표는 확정 진실이 아니라 현재 확인 가능한 근거에 기반한 운영용 메모다.
- 실제 티켓 상태는 별도의 이슈 트래커가 생기면 그곳을 기준으로 관리해야 한다.
- 새 작업이 시작되면 반드시 이 문서를 갱신한다.
- 저장소에 반영된 코드와 마이그레이션이 기준이며, 라이브 DB 반영 여부는 별도 메모로 관리한다.
