# 회의실 예약 시스템 개발 티켓

아래 티켓은 기획 기준으로 분해한 개발 작업 단위다. 구현 순서는 `P0` 우선이다.

## Epic A. 프로젝트 기초

### T-001. 프로젝트 초기 세팅
- 우선순위: P0
- 목적: Vercel + Supabase 기반 프로젝트 구조 준비
- 범위:
  - 프론트엔드 초기 구조
  - DB 연결 설정 기준 수립
  - 환경변수 체계 정리
- 완료 기준:
  - 개발/배포 환경이 분리되어 준비됨

### T-002. 서울시간 기준 정책 정의
- 우선순위: P0
- 목적: 모든 예약 판단을 `Asia/Seoul` 기준으로 통일
- 범위:
  - 시간대 기준 정책
  - 주간 범위 계산 정책
- 완료 기준:
  - 모든 예약 규칙이 서울시간 기준으로만 판단됨

## Epic B. 인증과 가입

- 우선순위: P0
