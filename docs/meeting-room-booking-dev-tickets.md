# 회의실 예약 시스템 개발 티켓

아래 티켓은 개발 유형별로 정리한 작업 단위다. 번호 체계는 `A-001`, `B-001`처럼 유형 문자 + 3자리 일련번호를 따른다.

## A. 인증 / 가입 / 승인

### A-001. 이메일 도메인 제한 가입
- 우선순위: P0
- 목적: 허용된 회사 이메일만 가입 가능하도록 제어
- 범위:
  - `samsung.com`
  - `partner.samsung.com`
- 완료 기준:
  - 그 외 도메인은 가입 불가

### A-002. 이메일 인증 플로우
- 우선순위: P0
- 목적: 가입 후 이메일 인증을 필수화
- 범위:
  - 인증 메일 발송
  - 인증 완료 상태 반영
- 완료 기준:
  - 인증 전 계정은 사용 불가

### A-003. 관리자 승인 플로우
- 우선순위: P0
- 목적: 이메일 인증 후 관리자 승인 절차 적용
- 범위:
  - 승인 대기 상태
  - 승인/거절 처리
- 완료 기준:
  - 승인된 사용자만 서비스 접근 가능

### A-004. 로그인 및 세션 유지
- 우선순위: P0
- 목적: 승인된 사용자의 로그인 상태 유지
- 범위:
  - 로그인
  - 로그아웃
  - 세션 유지
- 완료 기준:
  - 로그인한 사용자의 승인 상태가 유지됨

### A-005. 이메일 인증번호 발송 및 검증
- 우선순위: P0
- 목적: 가입 첫 단계에서 이메일 소유를 인증번호로 검증
- 범위:
  - 이메일 입력 단계
  - 10분 유효 인증번호 발송
  - 인증번호 검증
- 완료 기준:
  - 인증번호가 유효 시간 내에만 통과함

### A-006. 이메일 인증 실패 누적 잠금
- 우선순위: P0
- 목적: 같은 이메일의 인증번호 실패를 5회 누적 관리
- 범위:
  - 실패 횟수 누적
  - 재발송 시 누적 유지
  - 5회 초과 시 가입 차단
  - 관리자 화면에서만 잠금 해제
- 완료 기준:
  - 실패 5회 누적으로 해당 이메일 가입이 막힘

### A-007. 가입 후 이름, 비밀번호, 그룹 선택 단계
- 우선순위: P0
- 목적: 이메일 인증 후 이름, 비밀번호, 그룹을 입력받음
- 범위:
  - 이름 입력
  - 비밀번호 입력
  - 그룹 선택
  - `미지정` 그룹은 가입 화면에서 제외
- 완료 기준:
  - 인증 완료 후 그룹을 포함한 최종 가입이 가능함

### A-008. 관리자 거부 메일 발송
- 우선순위: P0
- 목적: 관리자 승인 거부 시 사용자에게 거부 안내 메일 발송
- 범위:
  - 거부 사유 메일 템플릿
  - 거부 상태 처리
  - 재신청 가능 상태 전환
- 완료 기준:
  - 거부된 사용자가 다시 1단계부터 재신청 가능함
