# 회의실 예약 시스템 전체 정책 문서

## 1. 서비스 목적
사내 사용자가 정해진 규칙 안에서 회의실을 예약하고, 관리자는 사용자 승인 및 회의실 운영을 통제할 수 있는 회의실 예약 시스템을 구축한다.

## 2. 서비스 범위
- 웹 프론트엔드
- 인증 및 회원가입
- 관리자 승인
- 회의실 관리
- 예약 생성 및 삭제
- 주간 예약 현황 대시보드
- 예약/삭제 이메일 알림
- 사용자 그룹 기반 통계

## 3. 기술 전제
- 프론트엔드: Vercel
- DB: Supabase Postgres
- 서버 기준 시간: `Asia/Seoul`

## 4. 사용자 가입 정책
- 가입 가능한 이메일 도메인:
  - `samsung.com`
  - `partner.samsung.com`
- 이메일 인증 완료가 필수다.
- 이메일 인증 이후 관리자 승인이 있어야 서비스 사용이 가능하다.
- 승인 전 사용자는 예약 기능을 사용할 수 없다.

## 5. 권한 정책
### 일반 사용자
- 로그인 가능
- 전체 예약 현황 조회 가능
- 본인 예약 생성 가능
- 본인 예약 삭제 가능
- 예약 생성/삭제 이메일 알림 수신

### 관리자
- 사용자 승인/거절 가능
- 회의실 생성/수정/삭제 가능
- 모든 예약 조회 가능
- 모든 예약 삭제 가능
- 사용자 그룹 관리 가능
- 통계 조회 가능
- 예약 정책 수정 가능

## 6. 회의실 정책
- 회의실은 이름과 숫자 등 식별 정보를 가진다.
- 회의실은 언제든 추가 가능하다.
- 회의실은 언제든 삭제 가능하다.
- 회의실 삭제 시 해당 회의실 예약도 함께 삭제한다.

## 7. 예약 정책
- 예약 가능 요일: 월요일부터 금요일
- 예약 가능 시간: 08:00 ~ 18:00
- 예약 단위: 30분
- 1회 최대 예약 가능 시간: 1시간
- 같은 회의실의 동일 시간대 중복 예약 불가
- 서버 기준 시간은 서울시간으로 통일

## 8. 주간 예약 제한 정책
- 사용자 1인당 주간 누적 예약 시간 제한을 둔다.
- 기본값은 3시간이다.
- 이 제한값은 관리자가 별도 설정 화면에서 수정 가능해야 한다.
- 예약 생성 시 해당 정책값을 기준으로 검증한다.

## 9. 예약 삭제 정책
- 예약 삭제는 예약자 본인 또는 관리자만 가능하다.
- 삭제 전에는 반드시 확인 팝업을 띄운다.
- 확인 후에만 최종 삭제된다.
- 삭제 시 예약자에게 이메일 알림을 발송한다.

## 10. 이메일 알림 정책
예약 생성 및 삭제 시 예약자에게 이메일 알림을 보낸다.

알림 대상:
- 예약자 본인

알림 시점:
- 예약 생성 직후
- 예약 삭제 직후

## 11. 사용자 그룹 정책
- 사용자 그룹은 권한 목적이 아니라 통계 목적이다.
- 관리자가 직접 그룹을 생성하고 사용자에게 배정한다.
- 예시:
  - 그룹 A
  - 그룹 B
  - 프로젝트 그룹
  - 임시 그룹
- 그룹별 예약 건수, 예약 시간, 사용 패턴 통계를 확인할 수 있어야 한다.

## 12. 대시보드 정책
### 사용자 대시보드
- 전체 회의실 예약 현황 조회 가능
- 본인 예약 목록 확인 가능
- 본인 예약 삭제 가능
- 주간 단위로 예약 상황을 파악 가능

### 관리자 대시보드
- 전체 예약 현황 확인
- 사용자 승인 관리
- 회의실 관리
- 사용자 그룹 관리
- 통계 확인
- 예약 정책 수정
